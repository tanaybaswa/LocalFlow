In plain English

LocalFlow lets you talk into any text field on your Mac and have the words appear — without sending your voice to the cloud. Hold the Right ⌘ key, speak, release. A local Whisper model turns speech into text and pastes it where you were typing.

That matters because everyday dictation tools often trade convenience for privacy: audio leaves your machine, and you depend on a network and a vendor. LocalFlow keeps audio on-device, runs at well above realtime on Apple Silicon, and behaves like a system utility rather than another chat window.

This document is also the handoff for the next developer: what shipped, why key decisions were made, where files live, and what to build next.

Why this is important

  • Privacy by default. Microphone audio is recorded, transcribed, and discarded locally. Nothing is uploaded for inference.
  • Works in any app. Slack, Cursor, browsers, Notes — wherever the caret is, the transcript pastes there.
  • Speed that feels native. On an M5 Max, Whisper large-v3 sustained roughly 12–17× realtime in file benchmarks — faster than speaking for typical clips.
  • Foundation for a product. The hard parts (warm local model, global hotkey, cross-app paste, permissions) are solved enough to iterate on UX rather than infrastructure.

What we built

For the person using it

  1. Launch LocalFlow from the Dock (menu bar waveform icon).
  2. Grant Microphone and Accessibility when asked.
  3. Wait until status says ready (Whisper loads once into a local server).
  4. Click any text field → hold Right ⌘ → speak → release.
  5. Text pastes in place. History lives in the Dock window. Cancel mid-hold with any other key.

While recording, a floating navy pill shows a live mint waveform driven by microphone amplitude (icon-matched colors). No “Listening” label — waveform only.

Performance evidence

File transcription throughput on Apple M5 Max (36 GB) with Metal-backed whisper.cpp and ggml-large-v3.bin. Higher × realtime means wall clock is a smaller fraction of audio duration.

Full methodology and raw timings live in that benchmark report. LocalFlow’s interactive path keeps whisper-server warm, so short utterances avoid cold-start cost that file CLI runs paid on every invocation.

How it works (engineering)

Bundle ID com.tanaybaswa.localflow. Swift / SwiftUI menu-bar app (SPM), ad-hoc signed .app via macapp/build-app.sh. Sandbox off — Accessibility, Automation, and a local HTTP sidecar require it.

Runtime pipeline

  1. Hotkey — HotkeyMonitor watches Right ⌘ (keycode 54). Hold starts capture; release finishes; other keys cancel.
  2. Focus capture — At hold start, PasteTarget.capture() stores the frontmost app and focused AX element (ignores LocalFlow itself).
  3. Audio — AudioRecorder taps AVAudioEngine, writes mono PCM16 WAV, and pushes RMS to AudioLevelMeter for the HUD.
  4. Indicator — RecordingIndicator is a borderless non-activating NSPanel drawn in pure AppKit (no NSHostingView gray fill). Bars from RecordingLevels (~30 fps).
  5. Inference — SidecarManager keeps Homebrew whisper-server on 127.0.0.1:12321. TranscriptionService POSTs multipart WAV to /inference.
  6. Paste — Single path only: put text on clipboard, restore captured app/AX focus, one System Events keystroke "v" using command down. No AX insert / CGEvent / multi-path fan-out (those caused triple-pastes and focus stealing).
  7. History — TranscriptStore records text, target app, paste method, and success for the Dock window.

Key decisions (do not casually reverse)

Decision Why Risk if reversed
Whisper large-v3 full GGML (~2.9 GB) Quality ceiling; still ≫ realtime on M5 Max Smaller models save disk but may hurt accuracy for product demos
Warm whisper-server sidecar Avoid cold load on every utterance Spawning CLI per utterance feels sluggish
One paste path (System Events ⌘V) Multi-path delivery triple-pasted into ChatGPT/Claude; AX prompt spam with ad-hoc builds Duplicate text; focus races; permission prompt fatigue
Capture paste target at record start Release may briefly focus LocalFlow or another window Paste lands in the wrong app
AppKit HUD, not SwiftUI hosting NSHostingView painted a gray rectangle around the rounded pill Visible chrome around the indicator
Right ⌘ only (not Left ⌘) Leaves Left ⌘ free for system shortcuts Conflicts with app menus / shortcuts

Disk, dependencies, overrides

Model file (the large artifact): models/ggml-large-v3.bin under the repo root (…/local-speech-to-text/models/) — about 2.9 GB. Downloaded via ./scripts/download-model.sh from ggerganov/whisper.cpp on Hugging Face. LocalFlow resolves it through AppPaths.modelURL (override with LOCALFLOW_MODEL / LOCALFLOW_ROOT).

Runtime deps: Homebrew whisper-cpp (whisper-server) and ffmpeg (server --convert). Gradio UI uses uv sync.

On a machine with ~1.6 TB free, 2.9 GB is not a capacity concern. Treat it as a known fixed cost of full large-v3 quality.

Permissions checklist

Permission Needed for If denied
Microphone Recording while holding Right ⌘ Cannot start capture
Accessibility Global hotkey + reading focused element Hotkey / target capture unreliable
Automation → System Events ⌘V paste into the target app Text stays on clipboard only

Ad-hoc signing means macOS may re-prompt Accessibility after rebuilds that change the code signature. Prefer stable signing for daily use, or re-toggle LocalFlow in System Settings → Privacy & Security → Accessibility.
